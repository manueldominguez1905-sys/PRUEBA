# Filosofía del Proyecto

**Estado:** Aprobado  
**Versión:** 1.0  
**Última actualización:** Julio 2026

---

# 1. Misión

Desarrollar una aplicación profesional para el conductor de TUSSAM que reúna en una única herramienta todas las funciones necesarias para la gestión diaria del trabajo.

La aplicación deberá ser rápida, fiable, intuitiva y preparada para evolucionar durante muchos años.

---

# 2. Visión

El proyecto evolucionará hasta convertirse en una plataforma integral capaz de gestionar:

- Diario del Conductor.
- Recaudaciones.
- Liquidaciones.
- Calendario de Turnos.
- Calendario Laboral.
- Hojas de Ruta.
- Nóminas.
- Convenios.
- Tablas Salariales.
- Estadísticas.
- Copias de Seguridad.
- Centro de Administración.

---

# 3. Principios

El desarrollo del proyecto seguirá siempre estos principios:

- Simplicidad antes que complejidad.
- Estabilidad antes que nuevas funciones.
- Reutilización antes que duplicación.
- Arquitectura antes que soluciones rápidas.
- Mantenimiento sencillo.
- Escalabilidad.
- Compatibilidad con versiones anteriores siempre que sea posible.

---

# 4. Filosofía de Desarrollo

Antes de desarrollar cualquier nueva funcionalidad deberán responderse estas preguntas:

1. ¿Aporta valor real al usuario?
2. ¿Respeta la arquitectura del proyecto?
3. ¿Puede reutilizar código existente?
4. ¿Mantiene el diseño visual de la aplicación?
5. ¿Es sencilla de mantener?

Si alguna respuesta es negativa, la funcionalidad deberá revisarse antes de desarrollarse.

---

# 5. Filosofía de Mantenimiento

Siempre tendrán prioridad:

1. Corrección de errores.
2. Estabilidad.
3. Arquitectura.
4. Rendimiento.
5. Nuevas funcionalidades.

---

# 6. Compromiso

El proyecto crecerá de forma ordenada.

Ninguna modificación importante se realizará sin documentarla previamente y sin verificar su compatibilidad con la arquitectura general.
